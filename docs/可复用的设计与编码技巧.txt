1. 计算合并调度（防抖式重算）
通过复用同一个 computing Promise，把同一轮的多次更新合并，避免重复排队和重入。
参考：runtime.js

2. 跨环境调度抽象
把 requestAnimationFrame、setImmediate、setTimeout(0)统一成一个 frame 入口，兼容浏览器和非浏览器环境。
参考：runtime.js

3. Promise 形式的失效钩子（你提到的这类技巧）
把 invalidation 设计成 Promise，并把 resolve 保存到变量的 _invalidate 上；重算或销毁时触发 resolve，实现“声明式清理钩子”。
参考：runtime.js, module.js, dispose-test.js

4. 特殊依赖的延迟注入
用保留符号（@variable / invalidation / visibility）作为占位输入，在真正执行定义函数前再替换成运行时对象，降低 API 表面复杂度。
参考：module.js, runtime.js

5. 同步/异步/生成器统一到一条 Promise 执行链
输入解析、定义执行、generator 首值拉取都挂在 Promise 链上，保证外部观察到一致的异步边界。
参考：runtime.js, generatorish.js, define-test.js

6. 版本号防陈旧写入
每轮计算携带 version token，完成时校验版本；过期结果抛 stale 哨兵，避免旧异步结果覆盖新定义。
参考：variable.js, module.js

7. 依赖图增量维护
通过 attach/detach 维护 inputs/outputs 有向边，并用 dirty/updates 集合只重算受影响子图。
参考：variable.js, runtime.js

8. 拓扑排序 + 环检测兜底
先按入度做拓扑执行，残留节点再做 circular 检测并统一抛循环定义错误，兼顾效率与可诊断性。
参考：runtime.js, define-test.js

9. 可达性驱动的懒计算
变量是否可达决定是否参与重算；不可达时触发失效，减少无观察者分支的计算和资源占用。
参考：runtime.js, define-test.js

10. 生成器生命周期管理

1. Promise 失效钩子（清理副作用首选）
用途：取消订阅、终止轮询、停止流、释放资源。
```js
function createInvalidation() {
  let invalidate = () => {};
  const invalidation = new Promise((resolve) => {
    invalidate = resolve;
  });
  return { invalidation, invalidate };
}

// usage
const { invalidation, invalidate } = createInvalidation();

async function runTask() {
  const sub = startSubscription();
  invalidation.then(() => sub.unsubscribe()); // cleanup hook
}

// later
invalidate(); // trigger cleanup
```

2. 版本号防陈旧写入（解决异步竞态）
用途：用户快速切换参数、重复请求返回乱序。
```js
let version = 0;
let value;

async function recompute(input) {
  const myVersion = ++version;
  const next = await fetchData(input);
  if (myVersion !== version) return; // stale result, ignore
  value = next;
}
```

3. 合并调度（批处理多次更新）
用途：高频状态变更时避免重复计算。
```js
let computing = null;

function scheduleCompute() {
  if (!computing) {
    computing = Promise.resolve().then(() => {
      computing = null;
      return computeNow(); // do batched work once
    });
  }
  return computing;
}
```

4. 拓扑执行 + 环检测（依赖图稳定方案）
用途：公式引擎、任务依赖、数据流节点。
```js
function runDag(nodes) {
  const queue = [];
  for (const n of nodes) if (n.indegree === 0) queue.push(n);

  let count = 0;
  while (queue.length) {
    const n = queue.pop();
    count++;
    n.run();
    for (const out of n.outputs) {
      out.indegree--;
      if (out.indegree === 0) queue.push(out);
    }
  }

  if (count !== nodes.length) {
    throw new Error("circular dependency");
  }
}
```

5. 两阶段构建（先建映射壳，再填充定义）
用途：模块派生、配置注入、跨环境重写依赖。
```js
function buildDerived(sourceGraph, overrides) {
  const map = new Map();

  // pass 1: create shells
  for (const src of sourceGraph.nodes) {
    map.set(src, { id: src.id, inputs: [], fn: null });
  }

  // pass 2: wire and copy definitions
  for (const src of sourceGraph.nodes) {
    const dst = map.get(src);
    dst.fn = overrides[src.id]?.fn ?? src.fn;
    dst.inputs = src.inputs.map((inp) => map.get(inp));
  }

  return [...map.values()];
}
```
